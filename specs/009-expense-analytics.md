# Spec-009: Expense Analytics Tool (`expense_summary`)

## 1. Overview
Currently, the Finance Intelligence MCP server contains a basic `summarize` tool that aggregates spending by category. However, it does not analyze trends over time (weekly, monthly, quarterly, yearly transitions), nor does it dynamically group by columns or generate visual charting.

This feature adds a separate analytics module (`analytics.py`) exposing a new MCP tool named `expense_summary` which aggregates expenses over dynamic dimensions (time periods and target columns) and uses **matplotlib** to generate graphical charts.

---

## 2. Requirements & User Stories

- **User Story 1**: As a user, I want to see how my spending changes over time: week-over-week, month-over-month, quarter-over-quarter, or year-over-year.
- **User Story 2**: As a user, I want to analyze my expenses grouped by another dimension, such as `category` or `subcategory`.
- **User Story 3**: As a user, I want a visual graphical chart (like a bar chart or pie chart) generated by a Python charting library and embedded directly in my LLM client response.
- **Visualization Strategy**:
  - The tool will use the `matplotlib` library to plot the analytical results.
  - The plots will be saved to a local folder in the workspace: `c:\Users\Admin\Desktop\Finance Intelligence MCP\charts\expense_analysis.png`.
  - The tool will return a Markdown image link pointing to this absolute file path (`file:///c:/Users/Admin/Desktop/Finance%20Intelligence%20MCP/charts/expense_analysis.png`), which is rendered natively by Cursor/Claude Desktop.
- **Security Constraint**: Users must only be allowed to summarize and plot their own transaction data.

## 3. Technical Design

### MCP Tool Interface
We will register a new tool using `@mcp.tool` in `main.py` and delegate the business logic to `analytics.py`.

```python
@mcp.tool
async def expense_summary(
    period: str = None,
    group_by: str = None,
    category: str = None,
    subcategory: str = None,
    start_date: str = None,
    end_date: str = None
) -> dict:
    """
    Generate an analytical expense summary with a rich Matplotlib chart.
    At least one grouping dimension ('period' or 'group_by') is recommended.
    
    :param period: Time aggregation: 'weekly', 'monthly', 'quarterly', 'yearly'.
    :param group_by: Column to group by: 'category', 'subcategory'.
    :param category: Optional filter by category.
    :param subcategory: Optional filter by subcategory.
    :param start_date: Optional filter to include expenses on or after this ISO date (YYYY-MM-DD).
    :param end_date: Optional filter to include expenses on or before this ISO date (YYYY-MM-DD).
    :return: A status dictionary containing the path to the generated chart image, markdown links, and tabular data.
    """
```

### Dependency Additions
We need to add `matplotlib` to `pyproject.toml`:
```toml
dependencies = [
    ...
    "matplotlib>=3.8.0",
]
```

### Aggregations & Query Construction
We will construct a dynamic group query based on `period` and/or `group_by`:
- **Period Truncations**:
  - `weekly`: `DATE_TRUNC('week', date)::date`
  - `monthly`: `DATE_TRUNC('month', date)::date`
  - `quarterly`: `DATE_TRUNC('quarter', date)::date`
  - `yearly`: `DATE_TRUNC('year', date)::date`
- **Group Columns**:
  - `category`: `category`
  - `subcategory`: `subcategory`

#### Query Template
```sql
SELECT 
    <select_clause> as dimension,
    SUM(amount) as total_amount,
    COUNT(id) as transaction_count
FROM expenses
WHERE user_id = $1 <AND_FILTERS>
GROUP BY dimension
ORDER BY dimension ASC
```

### Chart Generation Flow (Matplotlib)
1. Initialize a figure (e.g. `fig, ax = plt.subplots(figsize=(10, 6))`).
2. Plot the results based on the data:
   - For time-series (`period` is set): Plot a bar chart or line chart of dates/periods vs. `total_amount`.
   - For categorical groupings (`group_by` is set): Plot a horizontal/vertical bar chart or pie chart of categories vs. `total_amount`.
3. Format the chart professionally:
   - Apply clean background colors, labels, and legends.
   - Use grids and auto-format date labels.
4. Save the figure to `c:\Users\Admin\Desktop\Finance Intelligence MCP\charts\expense_analysis.png`. Ensure the parent folder `charts` is created.
5. Close the figure (`plt.close(fig)`) to release memory.

---

## 4. Edge Cases & Safety Constraints

- **No Expenses Found**: Do not generate a chart. Return a status dictionary stating no matches were found.
- **Multiple Groupings**: If both `period` and `group_by` are set, the SQL groups by both (e.g., monthly spending per category). In this case, plot a grouped bar chart or stacked bar chart.
- **Invalid Grouping Parameters**: Validate that `group_by` is in `['category', 'subcategory', None]` and `period` is in `['weekly', 'monthly', 'quarterly', 'yearly', None]`.
